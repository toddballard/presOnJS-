JavaScript Functions

Allright. Let's look at some function declarations and understand the differences between them. Remember that we're dealing with scope and context anytime we talk about functions. Also, remember that functions can be used as Constructors and that they always have Prototypes. 

							// Function Declaration
							function(){
								alert('foo');
							}

							// Named Function Declaration
							function foo(){
								alert('foo');
							}

							// Function Expression Declaration
							var foo = function(){
								alert('foo');
							};

							// Self-running Function Declaration
							(function(foo){
								alert(foo);
							})(foo);

JavaScript Objects

Let's look at Objects for a second. We need to understand how these Objects work in order to make them work for us. We're going to illustrate Objects, Object Literals, and Function Objects 

							// Object Declaration
							var coffee1 = new Object();
							coffee1.color = 'black';
							coffee1.taste = 'goog';
							coffee1.drink = function(){
								console.log('OMG coffee is good');
							};
							coffee1.drink();

							// Object Literal Declaration
							var coffee2 = {
								color:'black',
								taste:'good',
								drink:function(){
									console.log('OMG coffee is good.');
								}
							};
							coffee2.drink();

							// Function Object Declaration
							var Coffee = function(){
								this.color='black';
								this.taste='good';
								this.drink=function(){
									console.log('OMG coffee is good.');
								}
							};
							var coffee3 = new Coffee();
							coffee3.drink();

Now, from the way that these Objects look, you might expect that they behave in very similar ways. This is absolutely not the case as we will see when we run some further examples.

JavaScript Classes

Now that we know how classes are structured, we can build a function that will write this stuff for us. What we want to do is write a method on Function's Prototype that we can use anytime we want to create a sub-class. For this example, we'll call it makeSubClass(). 

							Function.prototype.makeSubclass= function() {
						    	function Class() {
							        if (!(this instanceof Class))
							            throw 'Constructor function requires new operator';
							        if ('_init' in this)
							            this._init.apply(this, arguments);
								}
							    if (this!==Object) {
							        Function.prototype.makeSubclass.nonconstructor.prototype= this.prototype;
							        Class.prototype= new Function.prototype.makeSubclass.nonconstructor();
							    }
							    return Class;
							};
							Function.prototype.makeSubclass.nonconstructor = function(){};

							var Person= Object.makeSubclass();
							Person.prototype._init= function(isDancing) {
							    this.dancing= isDancing;
							};
							Person.prototype.dance= function() {
							    console.log(this.dancing);
							};

							var Ninja = Person.makeSubclass();
							Ninja.prototype._init= function() {
							    Person.prototype._init.call(this, false);
							};
							Ninja.prototype.swingSword= function() {
							    console.log('Swipe swipe!');
							};

							var p= new Person(true);
							p.dance(); // => true

							var n = new Ninja();
							n.dance(); // => false
							n.swingSword(); // => true

Now, from the way that these Objects look, you might expect that they behave in very similar ways. This is absolutely not the case as we will see when we run some further examples.
